# Skill: Godot Scene Smoke Test (Headless)

## 何时使用

- 新增场景、改动场景绑定脚本、修改关键启动流程后。

## 目标

- 通过最小成本验证“场景可加载 + 脚本可解析 + 运行不立即报错”。

## 前置条件

- 已完成 `godot --headless --import`（如当前环境尚未导入）。

## 标准步骤

1. 对场景关联脚本做语法检查：
   - `godot --headless --check-only -s <script.gd>`
2. 运行目标场景：
   - `godot --headless <scene.tscn>`
3. 观察输出中是否存在解析失败、资源缺失、节点脚本绑定错误。

## 失败排查顺序

1. 脚本路径是否为 `res://` 且大小写一致。
2. 场景引用资源是否存在（贴图、音频、子场景）。
3. 节点导出变量类型是否与场景绑定一致。

## 完成标准

- 目标场景可被 headless 启动，且无启动阶段错误输出。
